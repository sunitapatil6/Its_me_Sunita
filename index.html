Subjects Taught (Semester Details)
3 Engineering Courses
Cloud Computing (CS401)
Code: CS401 Semester: VII Credits: 4 Lectures: 3 hrs/week

Course Objectives:

To introduce virtualization models, cloud storage mechanisms, container architectures (Docker/Kubernetes), cloud security paradigms, and SaaS/PaaS/IaaS setups using AWS and Azure.

Course Outcomes (COs):

  • CO1: Categorize virtualization models (Full, Para, OS-Level).
  • CO2: Deploy high-availability infrastructure deployments on Amazon AWS.
  • CO3: Optimize distributed cloud storage endpoints under REST paradigms.
  • CO4: Formulate robust orchestration architectures using Kubernetes.

Syllabus Overview:

  • Unit I: Introduction to Cloud & Systems Virtualization
  • Unit II: Cloud Architecture and Infrastructure Deployment models (IaaS, PaaS, SaaS)
  • Unit III: Cloud Storage Mechanisms (Block, Object, File systems - S3, EBS, GlusterFS)
  • Unit IV: Cloud Programming & Management (REST APIs, Microservices, Serverless)
  • Unit V: Containerization & Orchestration (Docker, Kubernetes pipelines)
  • Unit VI: Cloud Security, Auditing, and Fault Tolerance

Textbooks & Reference Books:

  • Textbook: Rajkumar Buyya, Christian Vecchiola, S. Thamarai Selvi, "Mastering Cloud Computing", McGraw Hill.
  • Reference: Barrie Sosinsky, "Cloud Computing Bible", Wiley Publishing.
Advanced Computer Architecture (CS402)
Code: CS402 Semester: VII Credits: 3 Lectures: 3 hrs/week

Course Objectives:

To discuss pipelines, parallel computing concepts, vector architectures, shared-memory systems, and instruction-level parallelism execution dynamics.

Course Outcomes (COs):

  • CO1: Evaluate quantitative computer performance laws (Amdahl's Law).
  • CO2: Resolve instruction data hazards in instruction scheduling pipelines.
  • CO3: Critique cache coherence issues in symmetric multiprocessor environments (Snoop/Directory).

Syllabus Overview:

  • Unit I: Fundamentals of Quantitative Design & Performance evaluation
  • Unit II: Instruction Level Parallelism (Dynamic Scheduling, Tomasulo's Algorithm)
  • Unit III: Hardware-Based Speculation & Branch Prediction Models
  • Unit IV: Thread Level Parallelism & Multiprocessors (Symmetric, Distributed Shared Memory)
  • Unit V: Cache Coherence Protocols (MESI, MOESI) and Snooping Architectures
  • Unit VI: Vector & GPU Architectures (SIMD computing frameworks)

Textbooks & Reference Books:

  • Textbook: John L. Hennessy and David A. Patterson, "Computer Architecture: A Quantitative Approach", Morgan Kaufmann.
  • Reference: Kai Hwang, "Advanced Computer Architecture: Parallelism, Scalability, Programmability", Tata McGraw Hill.
Distributed Systems (CS403)
Code: CS403 Semester: VII Credits: 4 Lectures: 3 hrs/week

Course Objectives:

To teach message-passing systems, consensus algorithms (Raft, Paxos), logical time clocks (Lamport, Vector), distributed databases, and high reliability architectures.

Course Outcomes (COs):

  • CO1: Implement RPC/RMI interfaces in network client-server sockets.
  • CO2: Apply clock synchronization standards to establish order of events.
  • CO3: Defend distributed transactions against deadlock anomalies.

Syllabus:

  • Unit I: Fundamentals of Distributed Systems and Architectures
  • Unit II: Communication Middleware (RESTful Web Services, RPC, Remote Method Invocation)
  • Unit III: Clock Synchronization & Logical Clocks (Lamport's Clocks, Vector Clocks)
  • Unit IV: Mutual Exclusion & Distributed Consensus (Raft Consensus, Bully Algorithm)
  • Unit V: Distributed File Systems (NFS, HDFS architectures)
  • Unit VI: Replication, Fault Tolerance & Recovery Mechanisms

Textbooks & Reference Books:

  • Textbook: Andrew S. Tanenbaum and Maarten Van Steen, "Distributed Systems: Principles and Paradigms", Pearson.
  • Reference: George Coulouris, Jean Dollimore, Tim Kindberg, "Distributed Systems: Concepts and Design", Addison-Wesley.

Academic FAQs & Rules
Syllabus & Code of Conduct
What is the attendance criteria to qualify for SPPU examinations?

As per SPPU and institute norms, a minimum of 75% attendance in both lectures and practical sessions is mandatory. Students falling below this threshold will be flagged for remedial tasks, and failure to comply may result in term work detention.

Can I get an extension on assignment deadlines?

Extensions are granted only in cases of documented medical emergency or official college representation. You must submit a request letter to Dr. Rajesh Sharma prior to the due date. Late submissions without approvals will face a 20% mark deduction penalty per day.

How are laboratory term-work marks calculated?

Term work marks (typically out of 25 or 50) are continuous assessments. They map: 40% on timely lab code validation, 30% on write-up completeness, 20% on weekly viva readiness, and 10% on attendance records.

What is the exam pattern for the Practical and Viva-Voce?

Practical exams consist of a 1-hour coding session based on random slips containing course assignments. This is followed by a 10-minute viva session by an external examiner covering theory definitions, code modifications, and course objectives.
